initializeActionOfMediaTimeUpdaterInAudioEditor

(
  • media
  • identifier
  • force_parsed_int
)

Defined in app/assets/javascripts/players.js:15

This method is fired each time the audio component player receives an event of timeupdate. There are two cases of use of this method:

  • if the Audio Editor is not in preview mode, it operates as a normal player (see initializeActionOfMediaTimeUpdater), with very slight differences due to the different environment
  • if we are in preview mode, the method acts as a counterpart of startAudioEditorPreview: at any time the seconds increase, the method updates the timer using increaseAudioEditorPreviewTimer; when the audio is over, if it's not playing the last component, the method calls startAudioEditorPreview for the following one.

Parameters:

  • media String

    HTML selector for the audio

  • identifier Number

    unique identifier for the audio component (see audio-editor)

  • force_parsed_int Boolean

    forces the last second of the audio; normally it's false, except the case in which the duration of the audio is very close to the biggest lower integer (for instance, the duration is 10.01, which is very close to 10): in this case, the signal that the audio ended is not received directly by the current method, but rather by the event handler ended initialized in the end of initializeAudioEditorCutter
