tags Module

Defined in: app/assets/javascripts/tags.js:1

The functions in this module handle two different functionalities of autocomplete for tags: suggestions for a research (search autocomplete), and suggestions for tagging lessons and media elements (tagging autocomplete). Both modes use the same JQuery plugin called JQueryAutocomplete (the same used in initNotificationsAutocomplete).

The search autocomplete mode requires a simple initializer (method initSearchTagsAutocomplete), which is called for three different keyword inputs of the search engine (the general one, the one for elements and the one for lessons).

The tagging autocomplete mode is slightly more complicated, because it must show to the user a friendly view of the tags he added (small boxes with an 'x' to remove it) and at the same time store a string value to be send to the rails backend. The implemented solution is a container div that contains a list of tag boxes (implemented with span, see createTagSpan) and an tag input where the user writes; when he inserts a new tag and presses enter or comma, the tag is added to the previous line in the container; if such a line is full, the tag input is moved to the next line; when the lines in the container are over, the tag input gets disabled (see disableTagsInputTooHigh). During this whole process, a hidden input gets updated with a string representing the current tags separated by comma (addToTagsValue, removeFromTagsValue).

The system also checks if the inserted tag is not repeated (using checkNoTagDuplicates), and assigns a different color for tags already in the database and for new ones (addTagWithoutSuggestion).

The tagging autocomplete mode is initialized using the scope class '_tags_container' which is unique (see method tagsDocumentReady).
