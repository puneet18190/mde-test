video-editor Module

Defined in: app/assets/javascripts/video_editor.js:1

The Video Editor is structured as follows: centered in the middle of the Editor is located the preview screen, below it the components timeline and the audio track, and on the right the preview column which contains also global statistics about the video.

A video created with the Video Editor can be composed by three types of components (and optionally an audio track):

  • a video component is an element of type video extracted from the user's gallery, associated to an initial and final time
  • a image component is an element of type image extracted from the user's gallery, associated to a duration in seconds (the image is held in the video for a number of seconds equal to the component's duration); the image is centered and cropped maintaining its original proportions, to make it fit in the video screen (which has proportions 16/9)
  • a text component is a centered title for which the user chooses a background color, a font color and a duration (which has the same interpretation as for image components).
The resulting video will be the concatenation of all the components inside the timeline, with optionally the chosen audio track as background audio. On the timeline the user may perform the following actions:

  • add a new component (see class VideoEditorAddComponents) or replace an existing one, even without maintaining its original type (see class VideoEditorReplaceComponents): these functionalities are initialized in videoEditorDocumentReadyAddComponent
  • remove a component from the timeline (initialized in videoEditorDocumentReadyRemoveComponent)
  • sort and change the order of the components (initialized in videoEditorDocumentReadyInitialization)
  • cut a video component (change its initial and final point) or change duration of an image or text compoent (both these functionalities are initialized in videoEditorDocumentReadyCutters and implemented in the class VideoEditorCutters).
Each component is provided of its own identifier (similar to the one used in audio-editor), that is unique and doesn't change on any operation performed by the user. Moreover, regardless of its type, a component is strictly linked with two external accessories:

  • a cutter (whose HTML id is video component [identifier] cutter): this item is normally hidden, when requested it appears below the timeline and is used to cut a video component or change the duration of an image or text component (class VideoEditorCutters)
  • a preview clip (whose HTML id is video component [identifier] preview): this item is hidden inside the preview screen, and it's used
    • to provide the user of a big clip of the component while handling it (functionality initialized in videoEditorDocumentReadyComponentsCommon)
    • to play a video component while cutting it (initialized in the method playersDocumentReadyVideoEditor in the module players)
    • to be shown in the global preview (see class VideoEditorPreview).
The method that extracts the identifier from a component is getVideoComponentIdentifier (it works receiving as parameter either the component or its cutter or preview clip).

While the user is working, the preview clip visible in the preview screen corresponds to the last component selected by the user. A component gets selected either if the user keeps the mouse on it for more than half a second (using the method startVideoEditorPreviewClipWithDelay, which avoids compulsive changes inside the preview screen when the user passes with the mouse over the timeline), or immediately on the actions of sorting and cutting (using the method startVideoEditorPreviewClip): both behaviors are initialized in videoEditorDocumentReadyComponentsCommon. To the preview clip of a video component is also associated a method (loadVideoComponentIfNotLoadedYet) that loads the HTML5 video tag only when necessary: this, similarly to audio-editor, avoids overloading of the HTML.

The component gallery used inside the Video Editor (initialized in initializeMixedGalleryInVideoEditor) is the only gallery in the application which contains mixed types of elements. It's divided into three sections, one for each kind of component: the sections for video and image components have the same external behavior of normal image and video galleries (see the module galleries), whereas the section for text components is a peculiar text editor (see the class VideoEditorTextComponentEditor). The component gallery (together with the regular audio gallery for the audio track) is initialized in videoEditorDocumentReadyGalleries, and its functionality defined in the methods of VideoEditorGalleries (for instance, the method to switch from a section to another).

The method videoEditorDocumentReadyAddComponent initializes the general procedure to add or replace a component. The system sets a HTML data that records if the component gallery was opened to replace or add a component: depending on this data, when the user picks a component from the gallery it's called the corresponding method in VideoEditorAddComponents or in VideoEditorReplaceComponents.

When the user adds a component, the system makes a copy of an empty hidden component and fills it with the new data. This behavior is quite similar to the one in audio-editor, but in the case of Video Editor the procedure is slightly more complicated, due to the following reasons:

  • there are three empty items (empty component, empty cutter, empty preview clip) for each type of component, for a total of nine
  • unlike audio-editor, in the Video Editor each component needs a miniature, that necessarily must be inserted in the empty component together with the rest of the data. For text components, the miniature is built in the moment of the component's creation (there is an empty text miniature hidden in the template of text component editor, see VideoEditorTextComponentEditor); for video and image components, the miniatures are loaded together with the mixed gallery and stored in the popup of each element (see module galleries, and especially the js.erb templates associated to the routes of the mixed gallery)
  • in the Video Editor it's possible to replace a component: when the system does this, it's not enough to fill the inputs of the previous component (with fillVideoEditorSingleParameter): it's additionally necessary to reset the inputs of the previous component, thing done by the method clearSpecificVideoEditorComponentParameters. Moreover, when replacing a component, the duration is updated using changeDurationVideoEditorComponent.
Besides the durations, two graphical details are peculiar to each component: the position, handled by reloadVideoEditorComponentPositions; and the transition, a small icon representing the fade transition of one second between a component and the following, that must be visible after all components except for the last one (see resetVisibilityOfVideoEditorTransitions). The operations in which callback it's necessary to reset transitions and positions are sorting (videoEditorDocumentReadyInitialization) and removing (videoEditorDocumentReadyRemoveComponent).

A video component cutter (or simply cutter) is an instrument used to change the initial and final second of a component of type video: it's very similar to the audio cutter, and its functionalities (JQueryUi sliders, players, etc) are defined in initializeVideoInVideoEditorPreview and playersDocumentReadyVideoEditor. A property that is worth mentioning is the automatic return to the previous integer second when pausing: this is a functionality of both cutters and global reproduction (see videoEditorDocumentReadyPreview and VideoEditorPreview), necessary to set with precision the current time of the preview screen, in order to simulate faithfully the effect of transitions and the correspondance with the optional audio track.

For image and text components, a cutter is simply a small form where the user may insert a new duration (the associated callback is changeDurationVideoEditorComponent. Since it doesn't fit the whole timeline, this paraticular cutter must be aligned to the JScrollPain: this is done with the functions of the class MediaElementEditorHorizontalTimelines.

All the cutters are initialized in videoEditorDocumentReadyCutters, and their functionalities included in the class AudioEditorCutters.

The text component editor can be opened clicking on the icon 'T' in the header of the component editor (see method switchToOtherGalleryInMixedGalleryInVideoEditor). The user can insert a text and choose background and text color: the functionality is initialized in videoEditorDocumentReadyTextComponentEditor, and the main methods are contained in the class VideoEditorTextComponentEditor. Notice that, unlike image and video components, the miniature of a text component is created in the moment of the insertion of the compoent (see both addTextComponentInVideoEditor and replaceTextComponentInVideoEditor).

The optional audio track is added clicking on the button on the bottom. When the user opens the audio gallery and selects an audio, the system automatically sets muted as true in all the video components (videoEditorDocumentReadyAudioTrack); when adding or replacing a video component (see both addVideoComponentInVideoEditor and replaceVideoComponentInVideoEditor) the attribute muted is inserted if an audio track is present.

The audio track is handled while playing a video component inside the video cutter (class VideoEditorCutters): the system ensures that the starting of the audio corresponds to the position of the selected video component inside the whole video, using the methods playersDocumentReadyVideoEditor and initializeActionOfMediaTimeUpdaterInVideoEditor of the module players.

Unlike the audio-editor, in Video Editor the preview mode is not left automatically when the user stops the global preview: The system has different behaviors for each item in the editor while being in preview mode. While the global preview is playing it sets to true the HTML data preview mode in-use.

While in preview mode, the preview button in the right column is substituted by a button play / pause; the arrow on the bottom used to commit changes (see mediaElementEditorDocumentReady) is replaced by a button 'X' that is used to leave the preview mode. Furthermore, in preview mode many graphical details are slightly changed respect to the normal mode (for instance it's not possible to open the cutters, remove components, etc).

The general methods relative to the preview mode are contained in VideoEditorPreview; the method to enter in preview mode is openPreviewModeInVideoEditor; the initializer, which contains also the functionality of leaving the preview mode, is videoEditorDocumentReadyPreview.

While in preview mode, it's possible to start the global preview (starting from the selected component at the selected second) using the method startVideoEditorGlobalPreview: this method is automatically fired with the first component selectedm when the user opens the preview mode for the first time. The method to play a component is playVideoEditorComponent; the functionality of passing from a component to another is placed in the last method (if the last component was of type image or text), and in initializeActionOfMediaTimeUpdaterInVideoEditor (it it's of type video, in which case the behavior differs slightly).

Unlike in audio-editor, there are two methods for increasing the global time: the regular one is increaseVideoEditorPreviewTimer (used to increase the time according to the currentTime of a video component), and the automatic one is automaticIncreaseVideoEditorPreviewTimer (used for automatic increase while playing image and text components).

Another important difference with audio-editor is the reproduction of the one second transitions between two components: if the user clicks on pause while the global preview is playing a transition, the method that is playing the component (as we have already seen, it can be either playVideoEditorComponent or initializeActionOfMediaTimeUpdaterInVideoEditor) waits until the transition is over to stop the reproduction.

In the class VideoEditorPreviewAccessories are stored the methods to handle the instruments available for the user to get to a specific position inside the timeline during preview mode. Such instruments (initialized in videoEditorDocumentReadyPreview) are:

  • the arrows that allow the user to pass from a component to another
  • the precision bar that allows the user to select a specific point inside the selected component: notice that this bar is implemented with a JQueryUi slider which is destroyed and re-initialized at each component change (using the methods showVideoEditorPreviewComponentProgressBar and hideVideoEditorPreviewComponentProgressBar).
Both precision instruments use the general method selectVideoComponentInPreview, that allows to select a component at a specific time.

To include the background audio track in the global preview, the system uses the method calculateVideoComponentStartSecondInVideoEditor, which calculates the total amount of seconds until the selected point, including the time necessary for the transitions.

Finally, let's have a look at the functionalities of the JScrollPain: the method followPreviewComponentsWithHorizontalScrollInVideoEditor is called at any time the selected component changes inside the preview mode (that is, either while reproducing the global preview, or if the user changes component using the arrows); this method re-implements many functionalities that were badly implemented or absent in the original plugin, such as for instance the uniform speed to scroll different amounts of components. The method uses the class MediaElementEditorHorizontalTimelines

As for the other Element Editors (image-editor, audio-editor) the core of the process of committing changes is handled in the module media-element-editor (more specificly in the class MediaElementEditorForms); the part of this functionality specific for the Video Editor is handled in mediaElementEditorDocumentReady.
